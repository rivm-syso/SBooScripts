# Setup for comparison

The below scripts are setup to compare a specific release to the current
version in use. E.g. loaded in R-Studio. (SBoo and SBooScripts). The
InstallSBoo function can be used to install a specific version to
compare to. This can be a release, e.g. the tag or specific branches in
SBoo and SBooScripts.

``` r
tag <- "2026.3.1"
InstallSBoo(Release = tag)
```

``` r
substance_current <- read.csv("data/Substances.csv")
substances_other <-  read.csv("SimpleBox/SBooScripts/data/Substances.csv")
```

## Calculate for installed version

Calculate first order rate constants for the other version. Here we
apply the new initWorldOther from the current branch as there are
missinng links to the new install folder in the old release. This can be
changed in future comparisons.

``` r
# Calculate the kaas 
other_kaas <- data.frame()
  
for(substance in substances$Substance){
 
  source("baseScripts/initWorldOther.R")
    
  other_kaas <- rbind(other_kaas,  World$kaas |>
    mutate(Substance = substance,
           ChemClass = World$fetchData("ChemClass")))
}
```

## Calculate for currently loaded version

Calculate the first order rate constants for the current branch

``` r
current_kaas <- data.frame()
  
for(substance in substances$Substance){

  source("baseScripts/initWorld.R")
    
  current_kaas <- rbind(current_kaas,  World$kaas |>
```

``` r
common_cols <- setdiff(intersect(colnames(current_kaas), colnames(other_kaas)), "k")

kaas_comparison <- merge(other_kaas, current_kaas, by=common_cols, suffixes = c("_Old", "_New"))

kaas_comparison <- kaas_comparison |>
```

# Specific’s on differences

No k’s have changed, which is to be expected, as shown:
`0 changed k's.`. In release 2026.4.0 the difference is in adding two
constants: Remove_global and Regional_and_Continental_deepocean. These
remove the Global compartments and turn Regional and Continental
deepocean on and off respectively. Because no other updates have been
done to SBoo, the k’s should not change when these variables are set to
FALSE, which is the default in InitWorld.R.

For more information about the differences between release 2026.3.1 and
2026.4.0 if these variables are true, see vignettes/Development/Quality
control/2026.3.1_comparison_with_Marine_currents.md.
